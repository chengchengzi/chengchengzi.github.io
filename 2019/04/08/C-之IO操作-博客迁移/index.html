C++之IO操作_博客迁移

coding
  • C++
  • IO输入输出
  • 原创

一直以来学IO都是有点蒙，理不清楚。最近码代码用到了有关文件读写的东西，正好回顾总结以下常用的一些东西，以便将来用到的时候方便查阅。

C++之IO操作

以下的内容查阅参考了：《C++primer》第四版以及C++的在线文档：http://www.cplusplus.com/reference/

C++使用标准库来处理面向流的IO操作

  • iostream处理控制台IO
  • fstream处理命名文件IO
  • stringstream处理内存string的IO

流的各种头文件

当需要用到的时候，直接#include最右边的头文件名就可以了。
处理|Input|Output|Header|
—:|:—:|:—:|:—:
控制台IO|istream(从流读取数据，如:cin)|ostream(向流中写入数据，如：cout/cerr/clog)|iostream(读写数据)
命名文件|ifstream(从一个给定文件读取数据)|osftream(向一个给定文件写入数据)|fstream(读写文件)
string|istringstream(从string读取数据)|ostringstream(向string写入数据)|stringstream(读写string)

具体用法

1、控制台IO

用得比较多的就是cin和cout了。cerr和clog都是ostream对象。cerr又叫标 准错误，一般用来输出警告和错误信息。clog用于产生程序执行的一般信息。

2、命名文件IO

（1）命名文件的读写有两者方式，分别是文本流(text streams) 和 二进制流（binary streams）；
（2）fstream的常用操作函数：

  • fsream file(s,mode); 创建一个fstream，并按制定的文件模式（mode)打开文件
  • file.open(s); 打开名为s的文件
  • file.close(); 关闭与file绑定的文件
  • file.close(); 检查是否成功打开文件
  • file.eof(); 检查是否到达文件尾
  • file.seekg(); 让文件指针定位到相应的位置
  • getline(); 读取一整行

(3）打开文件模式（mode)的有

mode 含义
in 以读方式打开
out 以写方式打开
app 每次写操作前均定位到文件末尾
ate 打开文件后立即定位到文件末尾
trunc 截断文件
binary 以二进制方式进行IO

(4) 例子：

  • 以文本流方式读写一个文件，读写的数据类型一般有string，int，double,float。而string可以和int，double，float相互转换，所有在这里统一用string数据类型来进行读写，当然也可以用stringstream来读写string。
  • int，float，double 转 string
  • string 转 int，float, double

  • 函数readAndWrite()先将str 写入文件中，再从文件读取显示
